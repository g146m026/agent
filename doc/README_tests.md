# README tests

Tous les tests ont été migrés et segmentés selon les standards :
- `qa/` : tests qualité et intégration
- `validate/` : validation de données, schémas, conformité
- `conformity/` : conformité aux standards internes/externes
- `standard/` : respect des conventions de code et d’API
- `best_practice/` : bonnes pratiques, sécurité, robustesse

Utiliser pytest pour l’exécution et la maintenance.
